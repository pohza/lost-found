# UAT Scenarios - Lost & Found

เอกสารนี้ใช้สำหรับ **User Acceptance Testing (UAT)** โดยจำลองสถานการณ์การใช้งานจริงของผู้ใช้ 2 กลุ่มหลัก:
- Finder (ผู้พบของ)
- Owner (เจ้าของของหาย)

## Scenario 1: Finder ลงประกาศของที่พบ และจัดการประกาศของตนเอง

**บทบาท:** Finder

### ขั้นตอน
1. Login เข้าใช้งานระบบ
2. ไปที่เมนู REPORT → เลือก Found
3. กรอกข้อมูลของที่พบ:
   - Item Name: "กระเป๋าสตางค์สีดำ"
   - Location: "ตึก A ชั้น 1"
   - Date & Time: วันที่/เวลา ที่พบ
   - Description: ระบุรายละเอียดภายในคร่าว ๆ
   - เลือกวิธีส่งมอบ: "ส่งให้จุดรับของกลาง"
   - วิธีติดต่อรับคืน: ระบุเงื่อนไขรับคืน
   - แนบรูปภาพ (ถ้ามี)
4. กด Submit
5. ไปที่หน้า My Posts ตรวจสอบว่าประกาศแสดงอยู่ในรายการ
6. จำลองสถานการณ์ "ของถูกส่งคืนแล้ว":
   - ใช้ปุ่ม Close เพื่อปิดประกาศ
7. ตรวจสอบว่าในหน้า Home ประกาศนั้นแสดงสถานะปิด (หรือไม่แสดงในรายการ active ตามการออกแบบ)

### ผลลัพธ์ที่คาดหวัง
- Finder สามารถลงประกาศได้สำเร็จ
- ประกาศของตัวเองแสดงใน My Posts
- สามารถปิดประกาศได้เมื่อของถูกส่งคืน

---

## Scenario 2: Owner ค้นหาประกาศ และส่ง Claim เพื่อขอรับของคืน

**บทบาท:** Owner

### ขั้นตอน
1. Login เข้าใช้งานระบบ
2. ไปหน้า Home
3. ใช้ Search และ Filter เพื่อค้นหาของที่หาย เช่น:
   - พิมพ์คำว่า "กระเป๋าสตางค์"
   - เลือกสถานที่ และช่วงวันที่ที่คาดว่าของหาย
4. คลิกประกาศที่คาดว่าน่าจะเป็นของตัวเอง → ไปหน้า Item Detail
5. กดปุ่ม Submit Claim
6. ในหน้า Claim กรอกข้อมูล:
   - Full Name, Contact Email, Contact Number
   - Description: อธิบายลักษณะเฉพาะของของ (เช่น จำนวนเงิน, บัตรอะไรบ้าง)
   - Serial Number (ถ้ามี)
   - แนบหลักฐานเป็นรูปภาพ (optional)
7. กด Submit Claim
8. ไปที่หน้า Claims Submitted เพื่อตรวจสอบว่าคำขอของตนเองถูกบันทึกแล้ว

### ผลลัพธ์ที่คาดหวัง
- Owner สามารถค้นหาประกาศที่เกี่ยวข้องได้จากหน้า Home
- สามารถส่ง Claim ได้ครบถ้วนพร้อมหลักฐาน
- คำขอปรากฏในหน้า Claims Submitted พร้อมสถานะเริ่มต้น (Submitted)

---

## Scenario 3: เจ้าของประกาศได้รับ Notification และตอบรับคำขอ

**บทบาท:** Finder (เจ้าของประกาศ)

### ขั้นตอน
1. Login เป็นผู้ใช้ที่เป็นเจ้าของประกาศ (Finder จาก Scenario 1)
2. ไปที่หน้า Notifications
3. เห็น notification แจ้งว่ามี Claim ใหม่เข้ามาสำหรับประกาศนั้น
4. คลิกรายการ notification → ไปหน้า Notification Detail
5. อ่านรายละเอียดจาก Owner:
   - Full Name
   - Contact Email / Number
   - Description ของหลักฐานความเป็นเจ้าของ
6. กดปุ่ม Approved (ถ้าข้อมูลถูกต้อง) หรือ Canceled (ถ้าไม่ตรง)
7. ตรวจสอบว่าหลังจากกดแล้ว ปุ่มถูก disable ชั่วคราวระหว่างโหลด และสถานะในระบบ (ตามที่ backend ออกแบบ) ถูกอัปเดต

### ผลลัพธ์ที่คาดหวัง
- เจ้าของประกาศเห็น notification ที่เกี่ยวข้องกับ Claim
- สามารถตัดสินใจอนุมัติหรือยกเลิกคำขอผ่านหน้า Notification Detail ได้

---

## Scenario 4: การสนทนาผ่าน Messages ระหว่าง Finder และ Owner

**บทบาท:** Finder + Owner

### ขั้นตอน (มุมมอง Finder)
1. Login เป็น Finder (เจ้าของประกาศ)
2. หลังจากมี Claim เข้ามา และ/หรือมีการ Approve/Canceled จาก Scenario 3
3. ไปที่หน้า Messages
4. เห็น thread ใหม่ที่เกี่ยวกับประกาศ/claim นั้น
5. คลิกเข้าไปที่ thread → หน้า Message Detail
6. พิมพ์ข้อความนัดหมายสถานที่/เวลารับคืน → กดส่ง

### ขั้นตอน (มุมมอง Owner)
1. Login เป็น Owner (ผู้ส่ง Claim)
2. ไปที่หน้า Messages
3. เห็น thread สนทนาเดียวกัน
4. ตอบกลับข้อความตามการนัดหมาย

### ผลลัพธ์ที่คาดหวัง
- ทั้งสองฝ่ายเห็นประวัติการสนทนาเดียวกันใน thread เดียว
- สามารถส่งข้อความโต้ตอบกันได้ โดยที่ข้อมูลติดต่อส่วนตัวไม่ถูกเปิดเผยใน public feed

---

## Scenario 5: การออกจากระบบอย่างปลอดภัย (Logout)

**บทบาท:** ผู้ใช้ทั่วไป

### ขั้นตอน
1. Login เข้าใช้งานระบบ
2. ไปที่หน้า Profile
3. กดปุ่ม Log out
4. ระบบพากลับไปหน้า Login
5. รีเฟรชหน้าเว็บ แล้วตรวจสอบว่าไม่สามารถเข้าหน้า Home/My Posts ได้หากไม่มีการ login ใหม่ (ขึ้นอยู่กับการ implement backend auth)

### ผลลัพธ์ที่คาดหวัง
- Token ถูกล้างจาก browser
- ผู้ใช้ต้อง login ใหม่ทุกครั้งหลัง logout เพื่อเข้าถึงหน้าที่ต้องการสิทธิ์
